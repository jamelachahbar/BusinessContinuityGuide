1Get Started in 60 seconds

BoltPlan runs entirely in your browser. No backend, no install. All data lives in localStorage, namespaced per solution.

  1. Clone & install git clone <repo> && cd web-app && npm install
  2. Run the dev server npm run dev — then open http://localhost:5173.
  3. Take the guided tour — it launches automatically on first run. Re-launch any time from the Tour button.
  4. Start with the Sample Solution to learn the patterns, then create your own with + New Solution.

A branded React Joyride walkthrough explains every section in context. Skip and re-run any time.

Tip Use the Tour button on the right side of the toolbar to re-launch the tour after configuring the app — it points to the exact features that ship now.

4Phase 1 — Prepare

Set the foundations: criticality tiers, business commitments, RACI, and test cadence. Maps to ISO 22301 Clauses 4–7.

Concepts Criticality Model Business Commitment Fault Model RACI Matrix Requirements Test Plans
Why start here? Your criticality tiers drive the BIA, the composite SLA targets, the recovery order, and the severity matrix. Get the model right before you touch architecture.

Exit checklist

  1. Criticality tiers approved by business sponsor.
  2. Business Commitment signed off by legal / product.
  3. RACI matrix populated with named individuals, not just role titles.
  4. Test cadence agreed.

6Build the Service Map

Drag-and-drop React Flow canvas with 100+ Azure services across 14 categories. Seven connection types let you express read/write, sync/async, control-plane, etc.

  1. Pick services from the Azure catalog drawer and drop them on the canvas. Don't see what you need? Select + Custom category… or + Custom service… in the dropdowns and type any free-form name (e.g. Mainframe, Oracle DB, SAP S/4HANA, On-prem AD).
  2. Connect them using one of 7 typed edges. Set direction, label, and color.
  3. Layout auto-snaps. Use the toolbar to fit-to-view and align.
  4. Export PNG for ADR documents or audit packages — the diagram is also embedded in the PDF/DOCX BCP.
Why React Flow? The graph is serialized to localStorage, so it round-trips through JSON export/import. Diagrams are first-class data, not screenshots.

7Run Fault Tree Analysis (IEC 61025)

Top-down deductive failure analysis using official symbols. Score each leaf with FMEA (Severity × Occurrence × Detection = RPN), then compare −BCDR and +BCDR.

FMEA dimensionScaleWhat it asks
Severity1–10How bad is the impact if this failure happens?
Occurrence1–10How often is it likely to happen?
Detection1–10How likely are you to catch it before customers do? (10 = very unlikely)
RPN1–1000S × O × D — prioritize anything ≥ 100.
Use it twice Build the tree under Assess with no mitigations (−BCDR). Re-score under Implement with your design choices in place (+BCDR). The delta is your reliability investment ROI.

Golden rule Write the runbook before the drill, not after. The runbook is the artifact under test. Every drill closes with a finding log + runbook diff.

11Operating Cadence

A monthly heartbeat plus an annual full cycle keeps the plan alive instead of shelfware.

WeekActivity
Week 1Review open findings from the last drill.
Week 2Walk one runbook with the on-call team.
Week 3Verify Composite SLA still matches the deployed architecture.
Week 4Export PDF + JSON. Attach to the monthly governance pack.
Annual full cycle BIA refresh → Architecture review → Full failover drill → BCP re-issue. Track each step in Phase 3 → Maintenance.
